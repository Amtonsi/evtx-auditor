# EVTX Auditor: инструкция пользователя

Разработал: Абдрахманов Амаль Даулетович

Лицензия: MIT

## Назначение

EVTX Auditor - настольное приложение для локальной проверки архивов с журналами событий Windows. Программа обрабатывает ZIP-архивы с журналами `.evtx` и legacy-журналами Windows XP / Windows Server 2003 `.evt`, находит критические и важные события, группирует повторы и формирует один сводный HTML-отчет.

Приложение работает локально. Журналы, архивы и результаты анализа не отправляются в сеть.

## Что поддерживается

- современные журналы Windows: `Application.evtx`, `Security.evtx`, `System.evtx`;
- legacy-журналы Windows XP / Windows Server 2003: `Application.evt`, `Security.evt`, `System.evt`;
- ZIP-архивы с журналами внутри папок узлов;
- один общий HTML-отчет с общей статистикой и отдельным разделом по каждому узлу;
- анализ последних 30 дней относительно самого позднего корректного события каждого узла;
- уровни Windows `Critical` и `Error`;
- события безопасности: очистка журналов, изменения политики аудита, пользователи, группы, блокировки, службы, Microsoft Defender;
- Windows XP EVT-события, включая Event ID 517, 529-539, 680, 624, 630, 642, 644, 632, 633, 636, 637, 660 и 661;
- массовые неудачные входы Event ID 4625: 10 и более попыток за 15 минут.

## Подготовка исходных данных

Рекомендуемая структура:

```text
C:\Users\impal\Downloads\АУДИТ 2
├─ NODE-01
│  ├─ logs_01.zip
│  └─ logs_02.zip
├─ NODE-02
│  └─ audit.zip
└─ NODE-03
   └─ windows_logs.zip
```

Внутри ZIP-архивов могут находиться файлы:

```text
Application.evtx
Security.evtx
System.evtx
Application.evt
Security.evt
System.evt
```

Если архив содержит другие файлы, приложение их игнорирует.

## Запуск готового приложения

1. Запустите `EVTXAuditor.exe`.
2. В поле "Папка с архивами" выберите каталог с архивами, например:

```text
C:\Users\impal\Downloads\АУДИТ 2
```

3. В поле "Папка для отчета" выберите каталог, куда нужно сохранить HTML.
4. Нажмите кнопку запуска проверки.
5. Дождитесь завершения анализа.
6. Откройте созданный HTML-отчет. Обычно он создается автоматически в выбранной папке.

## Как читать HTML-отчет

Отчет состоит из нескольких логических блоков.

### 1. Общая сводка

В верхней части отчета отображается:

- сколько узлов обработано;
- сколько узлов содержит находки;
- сколько архивов и журналов прочитано;
- сколько событий попало в период анализа;
- сколько критических, ошибочных и важных событий найдено;
- какие узлы имеют неполные данные или ошибки чтения.

### 2. Сводка по узлам

Для каждого узла показывается статус:

- `Есть находки` - данные обработаны, обнаружены проблемные события;
- `Проблем не найдено` - доступные данные обработаны, правила не сработали;
- `Неполные данные` - часть архивов, журналов или записей не удалось прочитать;
- `Ошибка обработки` - по узлу недостаточно пригодных данных для надежного вывода.

Статусы `Неполные данные` и `Ошибка обработки` нельзя трактовать как отсутствие проблем.

### 3. Раздел узла

Внутри раздела узла отображаются:

- имя узла;
- период анализа;
- список обработанных архивов;
- список журналов;
- найденные события;
- количество повторов;
- время первого и последнего события;
- Event ID, источник, уровень, канал и технические поля;
- диагностические сообщения по ошибкам чтения.

### 4. Группировка повторов

Если однотипная проблема повторяется много раз, отчет показывает ее как одну сгруппированную находку с количеством повторов. Это уменьшает объем отчета и сохраняет важные доказательные поля: время, Record ID, источник, Event ID и текст события, если он доступен.

## Особенности Windows XP EVT

Приложение читает `.evt` через классический Windows Event Log API. На современных Windows старые текстовые описания XP-событий могут быть недоступны, если на машине отсутствуют старые message DLL поставщика события.

В этом случае отчет не придумывает отсутствующий текст. Он сохраняет технические поля:

- Event ID;
- Source;
- Computer;
- EventType;
- RecordNumber;
- TimeGenerated;
- Category;
- строки события, если они доступны.

Правила анализа Windows XP работают по Event ID и техническим полям, поэтому отсутствие полного текстового описания не блокирует выявление типовых событий безопасности.

## Что считается критичным

В отчет попадают:

- Windows `Critical`;
- Windows `Error`;
- очистка журналов безопасности;
- изменение политики аудита;
- массовые неудачные входы;
- создание, удаление и изменение учетных записей;
- добавление и удаление участников групп безопасности;
- блокировка учетных записей;
- установка или изменение служб;
- события Microsoft Defender;
- ключевые события безопасности Windows XP.

## Ограничения

- Приложение анализирует только журналы, которые есть в архивах.
- Если журнал поврежден или архив неполный, отчет покажет диагностическое сообщение.
- Для `.evt` часть человекочитаемых описаний может быть недоступна без старых библиотек Windows XP.
- Отчет показывает признаки проблем по журналам, но не заменяет полноценное расследование инцидента.

## Сборка из исходного кода

Требования:

- Windows;
- Python 3.11 или новее;
- PowerShell;
- доступ к зависимостям из `requirements-dev.txt`.

Команды:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
.\.venv\Scripts\python.exe -m pip install -e .
powershell -ExecutionPolicy Bypass -File scripts\build.ps1
```

После сборки готовый файл находится в папке `dist`.

## Запуск из исходного кода

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
.\.venv\Scripts\python.exe -m pip install -e .
.\.venv\Scripts\python.exe -m evtx_auditor.main
```
